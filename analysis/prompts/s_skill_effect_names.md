あなたは「エイリアンのたまご」のゲーム仕様に精通した、高精度なデータ抽出エキスパートです。
あなたのタスクは、入力された**1つの特技説明文**を解析し、そこに含まれる「すべてのバフ・デバフ・状態異常関連効果」を抽出し、**JSON配列形式**で出力することです。

## 特技文の特徴

特技文は個性文と異なり、以下の特徴があります：
1. **基本ダメージ効果**: 「敵にダメージを与える」「敵に特大ダメージを与える」などは**特技の基本機能**であり、効果名として抽出**しません**
2. **発動タイミングが明確**: 「敵に～を与える」という形式が基本
3. **編成条件はほぼない**: 特技は編成条件（requirement）に依存しないことが多い
4. **追加効果**: 「さらに」「さらに一定時間」などの後に続く効果を抽出

## 抽出ルール

**最重要: 発動タイミング・条件・基本ダメージは絶対に無視し、追加効果のみを抽出してください。**

以下の項目は**完全に無視**してください：
- **基本ダメージ効果**: 「敵にダメージを与える」「敵に特大ダメージを与える」「敵に超ダメージを与える」「敵全体にダメージを与える」「敵小範囲に～ダメージを～回与える」など
- **時間条件**: 「○秒間」「一定時間」「○分間」など
- **確率表現**: 「することがある」「することもある」「させることもある」「高確率で」など（効果自体は抽出する）
- **敵依存条件**: 「敵がエイリアンの場合は」「敵が～の場合」など（これは条件分岐であり、効果自体ではない）

1.  **効果の網羅**: 特技文に含まれるバフ・デバフ・状態異常関連効果を**すべて**特定し、**効果1つにつき1オブジェクト** のJSONを生成してください。
    * **重要0: バフ・デバフの区別**: 
        * **バフ**: 味方に有利な効果（アップ、軽減、抵抗、回復など）
        * **デバフ**: 敵に不利な効果（ダウン、無効、状態異常付与など）
        * **自分へのデメリット（自己デバフ）**: 自分に不利な効果は除外
    * **重要1: 自分にデメリットになる効果は無視**: 自分にデメリット（減少・短縮・ダウン）として記述されている効果は無視してください。
    * **重要2: キラー系**: 
        * **特技ではキラー効果は使用されません**。特技文に「〇キラー」という記述があっても、これは効果名として抽出しないでください。
    * **重要3: 複数効果の分離**: 1つの特技文に複数の効果が含まれる場合は、すべて分離して抽出してください。
        * 例: 「敵に特大ダメージを与え、さらに一定時間、つよさをダウンし小人化するぞ！」→「つよさダウン」「小人化」の2つを抽出（「敵に特大ダメージを与え」「一定時間」は無視）
        * 例: 「敵に特大ダメージを与え、さらに6秒間、被ダメージを35％アップ！」→「被ダメージアップ」のみ抽出（「敵に特大ダメージを与え」「6秒間」は無視）
        * 例: 「敵にダメージを与え、毒状態や気絶状態にすることがあるぞ！」→「毒付与」「気絶付与」の2つを抽出（「敵にダメージを与え」「することがある」は無視）
    * **重要4: のけぞりと吹き飛ばし**: 
        * 「のけぞりやすくする」「のけぞりやすくなる」「のけぞらせやすくなる」→「のけぞり付与」
        * 「吹き飛ばす」「大きく吹き飛ばす」「敵を吹き飛ばし」→「吹き飛ばし」
    * **重要5: 無効化**: 
        * 「被ダメージ軽減バフを無効化する」→「被ダメージ軽減無効」
        * 「〇〇を無効化する」→ 該当する無効効果名を抽出（効果名の一覧に存在する場合）
    * **重要6: 効果名の表記**: 
        * 必ず下記の「効果名の一覧」に記載されている正確な表記を使用してください。
        * 例: 「HP回復」ではなく「たいりょく回復」、「やる気アップ」ではなく「やるきアップ」、「移動速度アップ」ではなく「いどうアップ」
2.  **抽出対象カラム**: 各効果について、以下のキーを抽出してください。
    * `name`: 効果名。**必ず下記の「効果名の一覧」から選択**してください。
    * `has_requirement`: 特技では**ほぼ常に `false`** です（編成条件がないため）。
    * `requirement_details`: 特技では**ほぼ常に空文字列 `""`** です。
    * `requirement_count`: 特技では**ほぼ常に `1`** です。
    * `target`: 特技では**無視して構いません**。空文字列 `""` または任意の値を出力してください。
    * `condition_target`: 特技では**無視して構いません**。空文字列 `""` または任意の値を出力してください。
3.  **完全に無視する項目**: 以下の項目は**この段階では一切考えない**でください。
    * 効果量、持続時間、発動確率
    * 発動タイミング
    * バフ枠の使用有無
    * 基本ダメージ効果
4.  **出力形式**: 必ず**JSON配列形式**で出力してください。該当する効果がない場合は、空の配列 `[]` を返してください。

## 特技タブでの分類とカテゴリ

メインアプリの特技タブ実装では、以下の分類構造とカテゴリが使用されています：

### 重要な原則

**効果名が決まれば、分類（回復系・バフ系・デバフ(状態異常)系）とカテゴリ（S_SKILL_HEAL・S_SKILL_BUFF・S_SKILL_DEBUFF）は一意に確定します。同じ効果名で異なる分類・カテゴリになることはありません。**

### 分類構造（categoryStructure）

特技タブでは以下の4つの分類に分類されます：

1. **ダメージのみ**
   - 基本ダメージ効果のみ（追加効果がない場合）

2. **回復系**
   - たいりょく回復
   - たいりょく吸収
   - 呪縛解除
   - 毒解除
   - 気絶解除
   - 足止め解除

3. **バフ系**
   - つよさアップ
   - ひきよせ
   - やるきアップ
   - クリティカルダメージアップ
   - クリティカル率アップ
   - ダメージ反射
   - 与ダメージアップ
   - 凶暴化
   - 回避率アップ
   - 奮迅
   - 攻撃力アップ
   - 攻撃回数アップ
   - 無双
   - 特技与ダメージアップ
   - いどうアップ
   - 被ダメージ軽減

4. **デバフ(状態異常)系**
   - たいりょく回復量ダウン
   - つよさダウン
   - やるきダウン
   - 攻撃力ダウン
   - いどうダウン
   - 急速ダメージ
   - 与ダメージダウン
   - 被ダメージアップ
   - 被ダメージ軽減無効
   - のけぞり付与
   - 吹き飛ばし
   - 呪縛
   - 小人化
   - 攻撃不可
   - 毒付与
   - 気絶付与
   - 足止め

## 解析対象

以下の特技テキストを解析し、JSON形式で出力してください:

### テキスト1
```
{skill_text1}
```

### テキスト2
```
{skill_text2}
```

### テキスト3
```
{skill_text3}
```

**出力形式:**
各テキストの解析結果を、テキスト番号（"1", "2", "3"）をキーとしたJSONオブジェクトで出力してください。
テキストが空（「なし」または空文字）の場合は、そのキーに空配列 `[]` を返してください。

```json
{
  "1": [
    {
      "name": "（効果名一覧から選択）",
      "has_requirement": false,
      "requirement_details": "",
      "requirement_count": 1,
      "target": "自分",
      "condition_target": ""
    }
  ],
  "2": [],
  "3": [...]
}
```

**重要**: 
- `has_requirement` は必ず boolean 型（`true` または `false`）で出力してください。特技では通常 `false` です。
- `requirement_details` は特技では通常空文字列 `""` です。
- `requirement_count` は必ず数値で出力してください（通常 `1`）。
- `target`: 特技では無視して構いません。空文字列 `""` または任意の値を出力してください。
- `condition_target`: 特技では無視して構いません。空文字列 `""` または任意の値を出力してください。
- **効果名の表記**: 必ず「効果名の一覧」に記載されている正確な表記を使用してください。
- 各テキストに対応する結果を番号（"1", "2", "3"）をキーとして必ず出力してください
- テキストが空や「なし」の場合、または効果が一切ない場合は、そのキーに空配列 `[]` を返してください
